Software Development Practices

Directory Structure and File Naming

The files in the GitHub repository are named clearly and consistently, and located in appropriately named directories. There is a clear separation between the student's files and any external files (for example, library files are all inside a directory named 'libraries'). You should award "Greatly Exceeds Expectations" on this criterion if the structure and naming is at the level you would expect from an experienced professional developer.

Version Control

The development was performed using git, with a commit for each meaningful change (rather than a single commit that covers everything). Commit messages are well-written.

Testing Implementation

All of the main functionality of the project has been tested using a suite of automated tests and/or well described manual testing scenarios (listed in either the readme or a separate file, or a link to a public page on an external tool, such as Trello). Consider awarding "Greatly Exceeds Expectations" if TDD practices were followed rather than having the tests written last-minute (check the commit log).

Testing Write-up

The testing planning, implementation, results and outcomes are described in detail, either in the readme, or a separate file. Interesting bugs discovered are discussed, and any known issues that were left unfixed (for whatever reason) are mentioned and explained.

Readme File

The project includes a readme file intended as an introduction for other developers who would like to use and contribute to this project. It should describe the project's components, the technologies used and any other important details. The file is formatted correctly using markdown (or another markup language) and is easy to read. Please note that the readme would usually contain content already assessed in other criteria (e.g. UX, testing, etc...), while this criterion is used for assessing that file as a whole.

Comments

All code files include clear and useful comments, wherever they relevant. These comments are consistent and explain the intent, reasoning and trade-offs behind different parts of the code, rather than just repeating what the code already says.

Deployment implementation

The project and all its dependencies (e.g. database, static resources) are running live on a hosting platform such as GitHub pages or Heroku without any issues. Please note that the free accounts on some hosting platforms like Heroku may take the site down at some hours, so allow for this possibility when grading. One way to get "Greatly Exceeds Expectations" on this criterion is by entirely automating all deployment procedures, such that the assessor would be able to set everything up on their own server by running a minimal set of commands.

Deployment Write-up
